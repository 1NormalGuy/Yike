# Jiang Yike's Secret - UI 内容规划说明

## 1. 产品定位

这是一个为 6 月 12 日生日准备的横屏互动礼物网页。它不是普通的品牌官网，也不是五个彼此独立的小游戏，而是一条完整的情绪体验：

**进入秘密世界 -> 完成五个挑战 -> 每关揭晓一份礼物 -> 集齐五份能量 -> 输入生日密码 -> 揭晓终极礼物。**

核心产品目标：

1. 让收礼人通过互动逐步发现礼物，而不是一次看完礼物清单。
2. 用“秘密、收集、召唤”的仪式感延长惊喜。
3. 五个小游戏负责制造参与感，礼物揭晓负责提供阶段性奖励，终极礼物负责情绪高潮。

核心关键词：

- 私人定制
- 浪漫生日
- 像素游戏
- 秘密礼物
- 收集与召唤
- 可爱、轻松、低挫败

## 2. 整体页面流程

```text
竖屏提示
  ↓ 转为横屏
开场页
  ↓ ENTER
礼物能量室（任务大厅）
  ↓ 选择任意游戏
五个小游戏之一
  ↓ 完成
礼物解锁弹层
  ↓ 自动返回
礼物能量室（进度增加）
  ↓ 五个全部完成
中央召唤阵解锁
  ↓
四位密码页
  ↓ 输入 0612
召唤动画
  ↓
终极礼物揭晓页
```

游戏进度保存在浏览器本地，用户中途退出后仍可继续。已完成的小游戏允许重玩，但不应再次制造“第一次发现”的误解。

## 3. 全局 UI 框架

### 3.1 使用场景

- 主要设备：平板横屏。
- 标准画布：16:9。
- 竖屏时不展示游戏内容，只展示旋转设备提示。
- 所有操作需要适合触控，按钮不能依赖鼠标悬停才能理解。

### 3.2 游戏页统一头部

小游戏统一使用三段式头部：

- 左侧：返回礼物能量室。
- 中间：关卡编号和游戏名称。
- 右侧：当前进度、分数或关卡数。

游戏画面中必须始终同时看见：

1. 当前要做什么。
2. 当前做到哪一步。
3. 成功或失败后的即时反馈。

### 3.3 全局状态

- 默认状态：尚未完成。
- 操作状态：可点击、观察中、输入中、动画中。
- 正确反馈：点亮、粒子、音效、文案。
- 错误反馈：抖动、变色、简短原因、允许立即重试。
- 完成状态：暂停游戏操作，进入礼物揭晓。
- 已完成状态：大厅中显示礼物和“已点亮，可重玩”。

## 4. 页面逐页内容逻辑

### 页面 0：竖屏提示页

**页面目标**

阻止用户在不适合的方向进入游戏，并明确下一步操作。

**内容层级**

1. 旋转设备图标。
2. 主标题：`请横屏游玩`。
3. 辅助文案：`转动平板，礼物召唤阵正在等你`。

**UI 重点**

- 页面只承担一个任务，不需要其他按钮。
- 图标应清楚表达“手机/平板旋转 90 度”，不能只依靠文字。
- 横屏后自动消失，不需要用户确认。

---

### 页面 1：开场页

**页面目标**

建立“这是专属于 Jiang Yike 的秘密生日体验”的第一印象，并引导进入。

**当前内容**

- 日期：`JUNE 12`
- 主标题：`JIANG YIKE'S SECRET`
- 核心悬念：`Five games. One secret.`
- 主按钮：`ENTER`
- 中央视觉：旋转的胶片/照片环。

**内容逻辑**

日期说明这次体验和生日有关；名字建立私人专属感；照片负责唤起共同记忆；一句话告诉用户需要完成五个游戏，但暂不揭晓最终礼物。

**UI 重点**

- 第一视觉应是“名字 + 私人照片”，不是礼物商品图。
- `ENTER` 是唯一主操作。
- 照片可以有动态，但不能影响标题和按钮识别。
- 第一次进入和完成后再次进入可以使用同一开场，但再次点击会直接回到终极揭晓。

**现有内容问题**

项目内已有“演唱会、登顶、雪山”三张照片，但程序当前只识别数字文件名，所以实际开场显示的是礼物商品图。新 UI 方案应明确使用真实照片，并统一照片裁切比例。

---

### 页面 2：礼物能量室 / 任务大厅

**页面目标**

让用户理解全部任务、自由选择游玩顺序、查看总进度，并在最终条件满足后进入召唤。

**内容层级**

1. 页面标题：`礼物能量室 / GIFT CHAMBER`。
2. 总进度：`能量收集 X / 5`。
3. 中央召唤阵：整个页面的最终目标。
4. 五个游戏入口：名称、简短玩法、完成状态。
5. 状态提示：完成礼物、还差几份能量、召唤阵已解锁。

**五个入口**

| 游戏 | 一句话玩法 | 完成后礼物 |
| --- | --- | --- |
| 星光接力 | 接住 6 颗星星 | 星黛露娃娃 |
| 愤怒小狗出击 | 弹射小狗击退黑猫 | Jellycat 金色小狗 |
| 治愈翻翻乐 | 翻牌找出相同图案 | Dior 护手霜 |
| 欧气盲盒机 | 记住并选中幸运盲盒 | 嘎子姐盲盒 |
| AI 心动便利贴 | 记住便利贴亮起顺序 | AI 心动便利贴 |

**入口状态**

- 未完成：礼物必须隐藏，只显示问号、游戏名和玩法。
- 已完成：展示真实礼物图、`已点亮`、`可重玩`。
- 刚完成返回：对应礼物高亮，并显示能量已加入召唤阵。

**召唤阵状态**

- 锁定：`能量未集齐`，点击后提示还差几份。
- 解锁：`召唤阵已解锁`，出现明确的进入按钮或点击提示。

**UI 重点**

- 大厅应首先让用户看懂“5 个任务围绕 1 个最终目标”，而不是五张平级菜单卡。
- 五个入口要有稳定编号、图标或位置，完成后不能重新排序。
- 当前画面存在中央文案与入口标签拥挤、底部说明偏小的问题，新方案应优先提高可读性。

---

### 页面 3：星光接力

**页面目标**

用最简单的操作完成第一种收集挑战。

**核心玩法**

- 左右拖动篮子。
- 接到星星：进度 `+1`。
- 接到炸弹：进度 `-1`，最低为 0。
- 收集 `6 / 6` 后完成。

**页面内容**

- 标题：`STAGE 01 / 星光接力`
- 进度：`星星 X / 6`
- 指令：`按住下方区域左右拖动篮子，避开小炸弹`
- 主游戏区：天空、掉落物、篮子。
- 独立触控区：`在这里左右拖动`

**UI 重点**

- 星星和炸弹必须有明显的形状与颜色差异。
- 篮子位置、可移动范围和触控区域要清楚。
- 收集反馈应即时，完成后停止继续掉落。

---

### 页面 4：愤怒小狗出击

**页面目标**

提供最有动作感的一关，用三关递进制造挑战感。

**核心玩法**

- 按住小狗向左下方拉动。
- 显示预测轨迹。
- 松手发射，撞击建筑并消灭黑猫。
- 共三关，黑猫数量依次为 1、2、3。

**页面内容**

- 标题：`STAGE 02 / 愤怒小狗出击`
- 进度：`关卡 X / 3`
- 关卡提示：`LEVEL X`
- 主游戏区：弹弓、小狗、轨迹、堡垒、黑猫。
- 辅助状态：剩余黑猫数量。
- 指令：按住、拉动、对准、松手。

**UI 重点**

- 用户必须看懂拉动方向，建议首次进入增加一段手势动画。
- 轨迹线、可发射状态和正在飞行状态要明显区分。
- 建筑是可破坏物，背景装饰不能比可交互对象更突出。

---

### 页面 5：治愈翻翻乐

**页面目标**

通过经典记忆配对玩法制造轻松、治愈的节奏。

**核心玩法**

- 每次翻两张卡。
- 相同则消除，不同则盖回。
- 三关规模依次为 4 对、6 对、8 对。

**页面内容**

- 标题：`STAGE 03 / 治愈翻翻乐`
- 进度：`关卡 X / 3`
- 左侧规则卡：玩法说明、剩余对数。
- 右侧卡牌棋盘。
- 底部动态提示：继续翻牌、配对成功、不相同。

**UI 重点**

- 卡背、翻开、配对成功、配对失败四种状态必须清晰。
- 规则说明只需首次快速理解，不能挤压主棋盘。
- 第三关有 16 张卡，需要提前按最密集状态设计。

---

### 页面 6：欧气盲盒机

**页面目标**

用“盯住目标”和“百里挑一”的夸张感强化幸运主题。

**三关逻辑**

1. 3 个盲盒：记住发光目标，经过 10 次交换后选择。
2. 5 个盲盒：同样玩法，但交换更快、次数更多。
3. 25 个盲盒：不再追踪目标，任意选择一个即可成功。

前两关均有幸运保底：连续两次未选中目标后，第三次选择必定命中。

**页面内容**

- 标题：`STAGE 04 / 欧气盲盒机`
- 进度：`关卡 X / 3`
- 顶部提示：记住目标、交换中、开始选择、选择错误、命中。
- 中央盲盒区域。
- 底部机器铭牌：`LUCKY GIFT MACHINE`。

**UI 重点**

- 第一、二关“目标预览”和“正在交换”必须有明确阶段差异。
- 交换结束前按钮不可点击，视觉上也要表现为观察阶段。
- 25 个盲盒组成清晰的 5×5 终极关卡，交换节奏放慢，并保证每个盒子容易辨认和点击。

---

### 页面 7：AI 心动便利贴

**页面目标**

通过记忆灯光顺序表达“AI 记住关于你的每一件小事”。

**核心玩法**

- 先观看便利贴按顺序亮起。
- 再按相同顺序点击。
- 错误后自动重播，不扣除生命。
- 三关分别为：4 张中记 3 次、4 张中记 4 次、6 张中记 5 次。

**便利贴内容**

- 心动
- 每天
- 早安
- 想你
- 晚安
- 陪你

**页面内容**

- 标题：`STAGE 05 / AI 心动便利贴`
- 进度：`关卡 X / 3`
- 左侧概念卡：心动备忘、玩法、本轮长度。
- 右侧便利贴区域。
- 顶部阶段提示：AI 正在写、请按顺序点击、顺序错误。
- 底部剩余点击数量。

**UI 重点**

- “系统演示中”和“轮到用户输入”必须非常清楚。
- 每张便利贴需要同时用颜色、图形、文字区分，不能只依赖颜色。
- 点亮顺序要有足够停留时间和声音反馈。

---

### 页面 8：通用礼物解锁弹层

**页面目标**

每完成一个游戏立即提供情绪奖励，并建立“礼物能量被加入大厅”的关系。

**展示顺序**

1. 深色遮罩和星光出现。
2. 卡片先显示问号。
3. 问号切换为真实礼物图。
4. 显示礼物名称。
5. 文案：`秘密礼物已加入召唤阵`。
6. 自动返回任务大厅。

**重玩状态**

- 标题改为“秘密礼物再次点亮”一类文案。
- 不应让用户误以为获得了第二份相同礼物。

**UI 重点**

- 这是每关的奖励时刻，礼物图和名称必须是绝对主视觉。
- 不要在弹层内加入过多说明或按钮。
- 揭晓节奏需要比普通成功 Toast 更隆重，但明显低于终极礼物。

---

### 页面 9：终极召唤密码

**页面目标**

在最终揭晓前增加最后一层私人记忆验证和仪式感。

**页面内容**

- 标题：`FINAL RITUAL / 终极礼物召唤`
- 状态：`法阵等待回应`
- 五份已解锁礼物能量围绕召唤阵。
- 问题：`输入属于这一天的四位召唤密码`
- 四位密码显示槽。
- 提示：`六月的特别日子`
- 数字键盘、删除、确认。

**密码逻辑**

- 正确密码：`0612`。
- 输入错误：面板抖动，密码清空，提示重新想日期。
- 输入正确：键盘禁用，五份能量依次汇聚，进入终极揭晓。

**UI 重点**

- 这里不能直接显示 `0612`，但提示要足够让生日本人理解。
- 数字键盘需要适合触控。
- 五份能量应该作为前面五个游戏成果的回顾，而不是新的信息。

---

### 页面 10：终极礼物揭晓

**页面目标**

完成整条体验的情绪高潮，明确现实礼物是什么。

**页面内容**

- 状态：`召唤成功`
- 标签：`LEGENDARY GIFT UNLOCKED`
- 终极礼物图：大疆 Pocket 4。
- 礼物名称：`大疆 Pocket 4`
- 文案：`召唤成功，大疆 Pocket 4 已解锁。请在现实世界接收你的礼物。`
- 背景音乐：《请和这样的我恋爱吧》。

**UI 重点**

- 最终页应比五个普通礼物的揭晓更克制、更聚焦、更有停留价值。
- 礼物图、礼物名、最后一句话构成唯一信息中心。
- 建议新增两个次级操作：`再看一次召唤`、`返回礼物房间`。当前版本没有明确退出路径。
- 再次进入网页时，如果已经完成，会直接回到此揭晓状态。

## 5. 礼物与游戏的内容关系

五个小游戏不是随机拼接，UI 可以加强玩法与礼物之间的象征关系：

| 游戏主题 | 礼物 | 可强化的设计联系 |
| --- | --- | --- |
| 星星、收集、可爱 | 星黛露娃娃 | 星光、紫粉色、梦幻角色感 |
| 小狗出击 | Jellycat 金色小狗 | 主角直接使用小狗形象 |
| 治愈、手部翻牌 | Dior 护手霜 | 护理、柔软、奶油质感 |
| 幸运盲盒 | 嘎子姐盲盒 | 机器、抽取、未知惊喜 |
| AI 记忆与陪伴 | AI 心动便利贴 | 备忘、早晚问候、陪伴感 |

## 6. 新 UI 必须交付的状态稿

不能只设计 8 张静态主页面，至少要覆盖：

1. 竖屏提示。
2. 开场页。
3. 大厅 0/5、部分完成、5/5 三种状态。
4. 每个小游戏的默认状态。
5. 游戏正确、错误、关卡完成的反馈规范。
6. 普通礼物揭晓弹层。
7. 密码默认、错误、召唤中三种状态。
8. 终极礼物揭晓。
9. 已完成后再次进入的状态。

## 7. 视觉和内容统一建议

### 保留

- 粉紫色私人生日氛围。
- 像素游戏语言。
- 英文小标题 + 中文主标题。
- 收集能量与召唤的世界观。
- 礼物揭晓前保持隐藏。

### 优化

- 像素字体只用于标题和标签，长中文说明改用更清晰的正文字体。
- 统一头部、进度组件、提示条、成功和失败反馈。
- 放大游戏说明和大厅底部文字，保证平板正常观看距离下可读。
- 降低背景装饰对可操作对象的干扰。
- 每页只保留一个最高优先级动作。
- 为首次操作增加动态手势提示，减少大段规则说明。

## 8. 当前版本需要 UI 特别解决的问题

1. 大厅五个入口和中央召唤阵信息有重叠感，层级不够稳定。
2. 多个页面使用小号像素中文，阅读成本偏高。
3. 开场应展示真实回忆照片，但目前错误显示了礼物图。
4. 游戏说明的位置不完全统一，有的在左侧卡片，有的在底部，有的在顶部。
5. 普通礼物解锁与终极礼物解锁的视觉结构相似，需要拉开仪式等级。
6. 最终揭晓后缺少返回、重播或结束动作。
7. 当前隐藏重置依赖连续点击五次，建议为维护者保留，但不要进入普通用户界面。

## 9. 给 UI 设计师的一句话 Brief

请把它设计成一场“为一个人准备的像素生日寻宝仪式”：用户始终知道下一步怎么玩，但永远要到完成挑战后才知道礼物是什么；每完成一关，房间更亮一点；五份惊喜最终共同指向 6 月 12 日和大疆 Pocket 4。
